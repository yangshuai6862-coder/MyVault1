# 一、适用场景

当出现以下任一情况，即启动本 SOP：

- ASIN前台页面属性中出现adult toys、adult sex toys等关键词
    
- ASIN 被标记为成人用品
    
- Listing 被限制展示 / 搜索不可见（自然位屏蔽最常见、这里需要注意浏览器cookie问题导致搜索位可见的情况：重新开一个无痕浏览器窗口再去搜索） **恶意篡改添加成人信息导致被屏蔽自然位逻辑：** 1.同行通过其他站点跟卖ASIN或者通过Amazon内部经理渠道修改ASIN详情页添加adult toys相关的内容包括但不限于：图片、详情页中部属性信息、五点描述等。 2.修改成功后一段时间内可能暂时不会被判定为成人用品，主要取决于亚马逊的系统刷新间隔（不固定时间，没有timely刷新），在被打上成人tag前，即没有被系统抓取前，如果及时清理掉了ASIN的所有成人信息，则可以避免自然位的消失。 3.如果未及时处理或处理后系统抓取更新的信息产生延迟，导致ASIN被判定为成人用品，打上成人tag，此时可以通过后台case-商品不能被搜索这条路径验证，显示被系统判定为成人用品。
    

# 二、第一时间处理流程

1.先开case，发送对应成人信息截图，要求客服查找对应ASIN在系统中的属性信息是否包含相应关键词 话术：请帮我查ASIN：（）全球成人用品信息，所有属性值。如果有成人信息的全部都要改掉，并且不要转受限商品团队。 查出来后问客服这些属性在哪些表格中，下载对应表格更新属性。 （一个asin 开一个case，开case的时候别说要解除成人用品标记或者搜索问题，就说需求就是改所有成人用品信息） 2.case查完，需要隔断时间再开case查询是否系统中还有相关属性，系统中是否显示已经针对这些属性就行了修改。 3.确认后用maxeyeAmazon邮箱发邮件到邮箱： pars-rp-core-sa-sep-transfer@amazon.com 要求来移除成人搜索受限标签。或者第一个case后也可直接发邮件。 Hello Support Team,

Seller merchant：A1VFUBKUUGHI0L （卖家记号） Store Name：Metapen Official （店铺名称）

ASIN：B09ZTXVNVD， US/AU/ES/IT/FR.

This product is not an adult product. Please help us remove the adult label so that we can be searched.

Best

（注意店铺卖家记号及名称的核对）

4.在被恶意篡改的一段时间内每天需要开case查询是否有成人信息、是否被标记为成人用品，如果有则需要及时按照前面的流程处理。
